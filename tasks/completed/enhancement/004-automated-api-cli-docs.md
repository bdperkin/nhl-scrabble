## Description

Extend the existing Diátaxis documentation framework by implementing automated API reference documentation generation from docstrings and automated CLI documentation generation from the Click interface. This provides machine-generated, always-up-to-date reference documentation that complements the manually-written tutorials, how-to guides, and explanations.

## Current State

The project has excellent foundation for automated documentation:

**Strengths:**

- ✅ 100% docstring coverage (enforced by interrogate pre-commit hook)
- ✅ Type hints throughout (strict mypy compliance)
- ✅ Well-structured Click CLI with comprehensive help text
- ✅ Diátaxis framework documentation structure in place
- ✅ Comprehensive manually-written documentation

**Current Documentation Structure:**

```
docs/
├── tutorials/          # Learning-oriented guides
├── how-to/            # Problem-oriented solutions
├── reference/         # MANUALLY-written technical reference

## Proposed Solution

Implement lightweight automated documentation generation tools that integrate with the existing Diátaxis structure:

### 1. Automated API Reference Documentation

Use **pdoc** or **pydoc-markdown** to generate API reference from docstrings:

**Option A: pdoc (Recommended)**

```bash
# Install
uv pip install pdoc

# Generate API docs
pdoc nhl_scrabble --output-dir docs/reference/api --format markdown
```

**Benefits:**

- Lightweight (no Sphinx needed)
- Generates clean markdown
- Integrates with existing docs/reference/ structure
- Auto-generates from docstrings
- Respects type hints
- Fast builds (\<1s)

**Option B: pydoc-markdown**

```bash
# Install
uv pip install pydoc-markdown

# Configure
pydoc-markdown --render-toc > docs/reference/api/README.md
```

**Benefits:**

- More customizable
- Supports custom templates
- Can generate single-file or multi-file output

### 2. Automated CLI Documentation

Use **sphinx-click** or **click-help-to-md** to generate CLI reference:

**Option A: click-help-to-md (Recommended)**

```python
# tools/generate_cli_docs.py
"""Generate CLI documentation from Click commands."""
import subprocess
from pathlib import Path

def generate_cli_docs():
    """Generate markdown CLI documentation."""
    output_file = Path("docs/reference/cli-generated.md")

    # Get CLI help text
    result = subprocess.run(
        ["nhl-scrabble", "--help"],
        capture_output=True,
        text=True,
        check=True
    )

    # Format as markdown
    content = f"""# CLI Reference (Auto-Generated)

*This file is automatically generated from the Click CLI. Do not edit manually.*

## Main Command

```

{result.stdout}

````

## Analyze Command

"""

    # Get analyze command help
    result = subprocess.run(
        ["nhl-scrabble", "analyze", "--help"],
        capture_output=True,
        text=True,
        check=True
    )

    content += f"""```
{result.stdout}
````

## Environment Variables

See [Environment Variables](environment-variables.md) for configuration options.

## Examples

See [Tutorials](../tutorials/) for usage examples.
"""

```
output_file.write_text(content)
print(f"✓ Generated {output_file}")
```

if __name__ == "__main__":
generate_cli_docs()

````

**Option B: cog + Click (Inline Generation)**

```python
# docs/reference/cli.md
# CLI Reference

<!-- [[[cog
import subprocess
result = subprocess.run(["nhl-scrabble", "--help"], capture_output=True, text=True)
cog.out("```\\n")
cog.out(result.stdout)
cog.out("```\\n")
]]] -->
<!-- [[[end]]] -->
````

Then run: `cog -r docs/reference/cli.md`

### 3. Documentation Build Automation

Add Makefile targets for documentation generation:

```makefile
# ============================================================================
# Documentation Generation Targets
# ============================================================================

.PHONY: docs-api
docs-api: ## Generate API reference documentation
	@echo "$(BLUE)Generating API reference documentation...$(RESET)"
	pdoc nhl_scrabble --output-dir docs/reference/api --format markdown --force
	@echo "$(GREEN)✓ API docs generated: docs/reference/api/$(RESET)"

.PHONY: docs-cli
docs-cli: ## Generate CLI reference documentation
	@echo "$(BLUE)Generating CLI reference documentation...$(RESET)"
	python tools/generate_cli_docs.py
	@echo "$(GREEN)✓ CLI docs generated: docs/reference/cli-generated.md$(RESET)"

.PHONY: docs-gen
docs-gen: docs-api docs-cli ## Generate all automated documentation
	@echo "$(GREEN)✓ All automated documentation generated$(RESET)"

.PHONY: docs-check
docs-check: docs-gen ## Check if generated docs are up-to-date
	@echo "$(BLUE)Checking if generated docs are up-to-date...$(RESET)"
	git diff --exit-code docs/reference/api/ docs/reference/cli-generated.md || \
		(echo "$(RED)✗ Generated docs are out of date! Run 'make docs-gen'$(RESET)" && exit 1)
	@echo "$(GREEN)✓ Generated docs are up-to-date$(RESET)"
```

### 4. CI Integration

Add pre-commit hook to ensure docs are up-to-date:

```yaml
# .pre-commit-config.yaml
repos:
  - repo: local
    hooks:
      - id: check-generated-docs
        name: Check generated documentation is up-to-date
        entry: make docs-check
        language: system
        pass_filenames: false
        always_run: true
```

Or add to GitHub Actions:

```yaml
# .github/workflows/ci.yml
jobs:
  check-docs:
    name: Check documentation
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: astral-sh/setup-uv@v4
      - run: make docs-gen
      - run: git diff --exit-code || (echo "Docs out of date" && exit 1)
```

### CI Integration

**Pre-commit Hook:**
- Hook ID: `check-generated-docs`
- Triggers: Changes to Python files or doc generator script
- Behavior: Gracefully skips if package not installed (pre-commit isolated env)
- Local: Runs check when package is installed (in venv)
- CI: Skips with warning, but tox environments validate docs

**Tox Integration:**
- All tox environments respect exclusions
- mdformat environment excludes cli-generated.md
- Generated API docs excluded from formatters

### Challenges Encountered

1. **Pre-commit Environment Isolation**
   - **Problem**: pdoc needs nhl_scrabble package importable
   - **Attempts**: Tried `language: python` with `additional_dependencies: ["."]` - failed
   - **Solution**: Made check_docs.sh detect if package is importable, skip gracefully if not
   - **Result**: Hook works locally (with venv), skips in CI pre-commit environment

2. **Mdformat Formatting Conflicts**
   - **Problem**: Generated CLI docs use different horizontal rule style than mdformat prefers
   - **First attempt**: Try to match mdformat's style in generator
   - **Solution**: Exclude cli-generated.md from mdformat in both .pre-commit-config.yaml and tox.ini
   - **Result**: Generated docs remain untouched by formatters

3. **Codespell False Positives**
   - **Problem**: Codespell finding "typos" in generated HTML (e.g., "nd" in HTML tags) <!-- codespell:ignore nd -->
   - **Solution**: Exclude docs/reference/api/ from codespell
   - **Result**: No false positives from generated docs

4. **CI Debugging Iterations**
   - 4 fix commits required to get all CI checks passing
   - Each iteration identified new edge cases
   - Final solution is robust and handles all environments
